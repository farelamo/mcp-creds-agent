# Vault Agent — MCP Server Sidecar

Automatic zero-downtime secret rotation for the MCP Multi-Tool Server running under Docker Compose.

---

## How It Works

```
Vault (remote)
    │  AppRole auth
    ▼
vault-agent container
    │  Renders → /run/secrets/<service>/.env  (tmpfs, RAM only)
    │  On change → docker kill --signal=SIGHUP mcp-server
    │
    ├─▶ /run/secrets/jenkins/.env
    ├─▶ /run/secrets/ssh/.env
    ├─▶ /run/secrets/ssh/id_ed25519
    ├─▶ /run/secrets/grafana/.env
    ├─▶ /run/secrets/elastic/.env
    ├─▶ /run/secrets/prometheus/.env
    ├─▶ /run/secrets/vault-mcp/.env
    └─▶ /run/secrets/custom/.env

MCP Server:
    • Reads credentials via mcpcreds.Store — re-reads on every call
    • go mcpcreds.WatchReload() — clears cache on SIGHUP
    • mcpcreds.Sanitize() — strips leaked creds from LLM output
```

## Quick Start

### 1. Bootstrap Vault (once)

```bash
export VAULT_ADDR=https://vault.your-domain.com:8200
export VAULT_TOKEN=<admin-token>
chmod +x scripts/bootstrap.sh
./scripts/bootstrap.sh
```

This enables AppRole auth, creates the `mcp-agent` policy, generates `role-id` / `secret-id`, and seeds Jenkins + SSH secrets interactively.

### 2. Configure

```bash
cp .env.example .env
# Edit .env with your Vault address
```

### 3. Start

```bash
docker compose up -d --build
```

### 4. Verify

```bash
docker compose ps
# vault-agent should be healthy
# mcp-server should be running
```

## Vault Secrets Layout

| Vault Path | Rendered To | Keys |
|---|---|---|
| `mcp/data/jenkins` | `/run/secrets/jenkins/.env` | `url`, `username`, `api_token` |
| `mcp/data/ssh` | `/run/secrets/ssh/.env` | `host`, `username`, `port` |
| `mcp/data/ssh-key` | `/run/secrets/ssh/id_ed25519` | `private_key` |
| `mcp/data/grafana` | `/run/secrets/grafana/.env` | `url`, `token` |
| `mcp/data/elastic` | `/run/secrets/elastic/.env` | `url`, `api_key` |
| `mcp/data/prometheus` | `/run/secrets/prometheus/.env` | `url` |
| `mcp/data/vault-access` | `/run/secrets/vault-mcp/.env` | `vault_addr`, `role` |
| `mcp/data/custom` | `/run/secrets/custom/.env` | any key-value pairs |

## Rotating Secrets (Zero Downtime)

```bash
# Rotate Jenkins token
vault kv patch mcp/jenkins api_token="new_token"

# Rotate SSH key
vault kv put mcp/ssh-key private_key="$(cat ~/.ssh/new_key)"

# Rotate Grafana token
vault kv patch mcp/grafana token="new_grafana_token"

# Rotate AppRole secret-id (Agent's own credential)
vault write -f -field=secret_id auth/approle/role/mcp-agent/secret-id \
  > ./agent/secret-id
docker compose restart vault-agent
```

Vault Agent detects the change, re-renders the `.env` file, and sends SIGHUP to the MCP server container. The `mcpcreds.Store` cache is invalidated and the next credential read picks up the new value.

## Adding a New Service

1. Store the secret in Vault:
   ```bash
   vault kv put mcp/my-service KEY=value
   ```

2. Add read access to `configs/mcp-policy.hcl`:
   ```hcl
   path "mcp/data/my-service" { capabilities = ["read"] }
   ```
   Then apply:
   ```bash
   vault policy write mcp-agent configs/mcp-policy.hcl
   ```

3. Create `templates/my-service.ctmpl`:
   ```
   {{- with secret "mcp/data/my-service" -}}
   MY_KEY={{ .Data.data.KEY }}
   {{- end }}
   ```

4. Add a `template {}` block to `configs/vault-agent.hcl`:
   ```hcl
   template {
     source      = "/etc/vault-agent/templates/my-service.ctmpl"
     destination = "/run/secrets/my-service/.env"
     perms       = "0600"
     command     = "docker kill --signal=SIGHUP mcp-server 2>/dev/null || true"
     error_on_missing_key = true
   }
   ```

5. In your Go code:
   ```go
   myStore := mcpcreds.New("my-service")
   ```

## File Layout

```
vault-agent/
├── .env.example                    # Vault address config template
├── agent/
│   ├── role-id                     # AppRole role ID (generated by bootstrap)
│   └── secret-id                   # AppRole secret ID (generated by bootstrap)
├── configs/
│   ├── vault-agent.hcl             # Agent config (auth, proxy, template blocks)
│   └── mcp-policy.hcl             # Read-only Vault policy for all services
├── scripts/
│   └── bootstrap.sh               # One-time Vault AppRole + secret seeding
└── templates/
    ├── jenkins.ctmpl              # Jenkins credentials
    ├── ssh.ctmpl                  # SSH connection params
    ├── ssh-key.ctmpl              # SSH private key (rendered as file)
    ├── grafana.ctmpl              # Grafana credentials
    ├── elastic.ctmpl              # Elasticsearch credentials
    ├── prometheus.ctmpl           # Prometheus credentials
    ├── vault-mcp.ctmpl           # Vault access for the MCP vault tool
    └── custom.ctmpl              # Generic key-value (renders all keys dynamically)
```

## Vault Agent Proxy

The agent exposes a local proxy at `http://vault-agent:8007` with auto-auth token injection. The MCP server's Vault tool uses this via `VAULT_AGENT_ADDR` so it never needs a Vault token directly.

## Credential Library

This setup uses [github.com/farelamo/mcp-creds](https://github.com/farelamo/mcp-creds) — a Go library for zero-trust credential loading in MCP servers:

```go
import mcpcreds "github.com/farelamo/mcp-creds"

// Hot-reload on SIGHUP from Vault Agent
go mcpcreds.WatchReload(mcpcreds.Jenkins, mcpcreds.SSH)

// Read fresh credentials (cached 2s, invalidated on rotation)
token, err := mcpcreds.Jenkins.Require("JENKINS_TOKEN")

// Sanitize output before it reaches the LLM
result := mcpcreds.Sanitize(apiResponse)
```

See the [mcp-creds README](https://github.com/farelamo/mcp-creds) for full API and integration examples.
